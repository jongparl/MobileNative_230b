System Information Tutorial

This tutorial demonstrates how to use the System Information API. The functions contained in this API allow you to obtain information about the device, such as platform versions, device model, and supported device features.

API functions and data types are defined in the system_info.h header. This header file must be included in all applications that use the System Information API.

To learn about the System Information API features, see the System Information Programming Guide.

This tutorial consists of the following parts:

  • Getting the Device Model Name
  • Checking Whether a Camera is Provided

Getting the Device Model Name

System information consists of "key, value" pairs. To get the device model name, use the http://tizen.org/system/model_name key.

The type of the value is String, which means that the system_info_get_platform_string() function is used for the information. The value of the key must be released by the free() function.

#include <system_info.h>
void func(void)
{
   char *value;
   int ret;

   ret = system_info_get_platform_string("tizen.org/system/model_name", &value);
   if (ret != SYSTEM_INFO_ERROR_NONE) 
   {
      // Error handling
      return;
   }

   dlog_print(DLOG_INFO, LOG_TAG, "Model name: %s", value);

   free(value); // Release after use 
}

Checking Whether a Camera is Provided

To check whether the device provides any kind of a camera, use the http://tizen.org/feature/camera key.

The type of the value is bool, which means that the system_info_get_platform_bool() function is used for the information.

#include <stdbool.h>
#include <system_info.h>

void func(void)
{
   bool value;
   int ret;

   ret = system_info_get_platform_bool("tizen.org/feature/camera", &value);
   if (ret != SYSTEM_INFO_ERROR_NONE) 
   {
      // Error handling
      return;
   }

   dlog_print(DLOG_INFO, LOG_TAG, "Camera: %s", value ? "Supported" : "Not supported");
}
