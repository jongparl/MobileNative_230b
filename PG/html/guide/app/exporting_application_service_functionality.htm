Exporting Application Service Functionality

You can export functionalities of your application to be used in another application. However, if the other application uses the application service functionality implicitly without the application ID, it must declare the application service information in its manifest file:

<application-service>
   <mime name="application/xhtml+xml"/>
   <operation name="http://tizen.org/appcontrol/operation/view"/>
   <uri name="http://test.com"/>
</application-service>
<application-service>
   <operation name="http://tizen.org/appcontrol/operation/call"/>
</application-service>

The resolution filter is used when resolving the application service. Each entry of the resolution filter has the <application-service> element and forms an application resolution unit. The additional URI or MIME type information must contain the associated operation ID, and the retrieved application service must have the specified application ID and operation ID when resolving the application service.

You can specify the application service information of your application in the application project settings in the IDE; an operation ID must be specified for the application service.
