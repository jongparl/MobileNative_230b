WebView

A significant amount of user-controlled online interaction is done with a Web browser instead of direct interfaces to Web resources. This makes the Web browser an important part of applications that have online functions.

EFL WebKit (EWK) is an API set for building Web client applications, such as a Web browser. EWK provides various features for Web browsing, including loading and displaying Web pages, and navigating through the browsing history.

With the WebView API, you can create a simple Web browser with minimal functionality. The EWK Browser_window structure allows the user to create and initialize windows, display Web pages, and navigate through the browsing history.

The following figure illustrates the EWK Browser_window structure.

Figure: EWK Browser_window structure

The structure has the following elements:

  • evas object

    This object is a basic, clean display canvas API for several target display systems, and it can draw, for example, smooth super and sub-sampled images, and anti-aliased text. It is used to create areas or windows that the application can draw on with an X Window System.

    A smart evas object provides custom functions to handle, for example, clipping, hiding, moving, and resizing of child elements automatically. Smart objects are defined by smart classes, which are structs defining their interfaces or smart functions. The base class for any smart object is Evas_Smart_Class.

  • Ewk_Context

    This object encapsulates all pages related to a specific use of WebKit. The context can be the default one or created by the application.

  • Smart event

    This object is an event elected by a smart object and occurring inside it. Smart events report back.

  • Ewk_view

    This object is the high level access to WebKit-EFL browser component. It manages the main frame and other critical resources.
