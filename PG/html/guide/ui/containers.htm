Container Widgets

Elementary provides various container widgets that can be directly used in the user interface of the application and contain other widgets. This guide describes the following container widgets:

  • Layout
  • Box
  • Conformant
  • Naviframe
  • Scroller
  • Panes
  • Table

Layout

Layout is a container widget, which takes a standard Edje design file and wraps it very thinly in a widget.

An Edje design theme file defines how the UI elements are positioned and how they behave upon actions. For more information, see Edje API Reference.

The layout widget can contain as many parts and children as described in its theme file. The children can have special types:

  • SWALLOW (content holder)

    Only one object can be added to a SWALLOW type. The elm_layout_content_set/get/unset() functions can be used to manage objects in a SWALLOW part. After being set to this part, the object size, position, visibility, clipping, and other description properties are controlled by the description of the given part (within the Edje theme file).

  • BOX

    The BOX layout can be used through the elm_layout_box_*() set of functions. It is very similar to the elm_box widget but the BOX layout behavior is controlled by the Edje theme.

  • TABLE

    The TABLE layout is similar to the BOX layout, but it is used through the elm_layout_table_*() set of functions.

The layout widget provides the following predefined themes:

  • toolbar-content: Application with a toolbar and main content area
  • toolbar-content-back: Application with a toolbar and main content (with a back button) and title areas
  • toolbar-content-back-next: Application with a toolbar and main content (with back and next buttons) and title areas
  • content-back: Application with main content (with a back button) and title areas
  • content-back-next: Application with main content (with back and next buttons) and title areas
  • toolbar-vbox: Application with a toolbar and main content area as a vertical box
  • toolbar-table: Application with a toolbar and main content area as a table

You can set the themes to a layout using the elm_layout_theme_set() function.

Box

The box widget arranges objects in a linear fashion vertically or horizontally. You can configure the padding and alignment using the elm_box_align_set() and elm_box_padding_set() functions.

Figure: Box container

Create the box container using the elm_box_add() function, and add objects using the elm_box_pack_end(), elm_box_pack_start(), elm_box_pack_after(), and elm_box_pack_before() functions.

Box objects can be organized in different ways by customizing or creating new layout functions. The elm_box_layout_set() function can be called to set a new layout function.

The elm_box_layout_transition() function provides the ability to animate the motion of the children in a box when switching from one layout to another.

Conformant

Conformant is a container widget that accounts for space taken by the indicator, virtual keyboard, and softkey windows. The conformant content is resized and positioned based on the available space

Figure: Conformant container

Create a conformant using the elm_conformant_add() function and set its content using the elm_object_content_set() function.

You can detect the following signals:

  • virtualkeypad,state,on: Virtual keypad state changed to on.
  • virtualkeypad,state,off: Virtual keypad state changed to off.
  • virtualkeypad,size,changed: Virtual keypad size changed.
  • clipboard,state,on: Clipboard state changed to on.
  • clipboard,state,off: Clipboard state changed to off.

Naviframe

The naviframe widget consists of a stack of views. New items are pushed on top of the previous one, and only the topmost view is displayed. When a new item is pushed, the previous items are not deleted. They are shown when the upper view is popped. Transitions can be animated on a push or a pop, depending on the theme applied to the widget.

Figure: Naviframe container

Naviframe views contain various elements:

  • Back button, used to navigate to previous views
  • Next button, used to navigate to next views in the stack
  • Title label
  • Subtitle label
  • Title icon
  • Content area

Use the elm_object_item_part_content_set() function to set the content in the naviframe with the following part names:

  • default: Main content of the current page
  • icon: Icon in the title area of the current page
  • prev_btn: Button of the current page to go to the previous page
  • next_btn: Button of the current page to go to the next page

Text can be set using the elm_object_item_part_text_set() function:

  • default: Sets the title label in the title area of the current page.
  • subtitle: Sets the subtitle label in the title area of the current page.

New views are pushed using the elm_naviframe_item_push() function. Item style can be modified to change the transitions between the views using the elm_naviframe_item_style_set() function.

Naviframe emits the following signals:

  • transition,finished: Transition is finished in changing the item
  • title,transition,finished: Title area transition is finished in changing the title state
  • title,clicked: User clicked the title area

Scroller

The scroller widget provides a way to scroll an Elementary object. A scroller is used in a many widgets (such as list, genlist, and map), and allows the user to scroll the content with a scrollbar or to drag the object around with a finger.

Several functions can be called on this object. Bouncing behavior is enabled using the elm_scroller_bounce_set() function; a specific region of the scroller content can be brought in using the elm_scroller_region_bring_in() function; and gravity can be configured using the elm_scroller_gravity_set() function.

Panes

The panes widget adds a draggable bar between content areas. The contents are resized when the bar is dragged.

Figure: Panes widget

Add a pane using the elm_panes_add() function, and add content to the left part of the panes:

elm_object_part_content_set(panes, "left", obj);

Pane orientation can be set using the elm_panes_horizontal_set() function, and its size can be set using the elm_panes_content_left_size_set() and elm_panes_content_right_size_set() functions.

The widget emits the following signals:

  • press: Pane area has been pressed (button has not been released).
  • unpress: Pane area was released after being pressed.
  • clickedv: Pane area has been clicked.
  • clicked,double: Pane area has been double-clicked.

Table

The table widget can contain several items that span on multiple rows or columns. The number of rows and columns is adjusted by the table when items are added to it.

Figure: Table widget

Create the table using the elm_table_add() Function, and add items to it using the elm_table_pack() function. Padding between rows and columns can be configured using the elm_table_padding_set() function.
