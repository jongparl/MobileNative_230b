UI: Creating the Application UI

The UI API provides the functionality to create and manage your application user interface. The heart of the Tizen application UI is the native UI framework, which consists of the Enlightenment Foundation Libraries (EFL).

Figure: EFL

The main features of the UI API include:

  • EFL

    EFL contains a set of layered libraries, and functions as the window manager for your application.

  • Eina

    The Eina library is part of EFL, and provides APIs for creating and manipulating various data types, such as arrays, lists, and strings. The Eina library also provides a set of tools for, for example, error management, type conversion, and memory pool management.

  • Evas

    The Evas library is part of EFL, and forms a clean display canvas for the target display systems, allowing you to manipulate objects on the canvas.

    Evas also provides UI rendering using the retained mode.

  • Elementary

    The Elementary library is part of EFL, and provides the basic toolkit for window and layout management, along with the UI and container widgets for the user interface.

  • Event handling

    The user interface relies on callbacks, designed to react to EFL events generated by the system or the user. You can handle various events, from low-level Ecore system events to Evas smart object events that happen on object collections.

  • EFL UTIL

    The ELF UTIL library supports additional EFL features, such as managing notification window levels.

  • TBM surface

    The Tizen Buffer Manager (TBM) surface provides functions for the graphic buffer in Tizen.
