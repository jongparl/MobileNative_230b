Package ID and Application ID

When developing a Tizen application, you must define a package and application ID:

  • Package ID

    The package ID, such as com.samsung.message or com.samsung.call, is the application package identifier.

    You can use the package ID as a directory name to create a unique property, such as /opt/usr/apps/com.samsung.message/bin.

    The package ID is functions as a unit for package installation, uninstallation, and upgrades. It is also the unit of the application package DRM, sandboxing, and privilege.

  • Application ID

    The application ID (appid) is the application identifier. Multiple applications are allowed in a package and the application ID is used to identify the different applications.

    The application ID can be the same as the package ID, such as com.samsung.message or com.samsung.message.consumer.

    The application ID functions as a unit for launching and terminating applications, and for App Control.
