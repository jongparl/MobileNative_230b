Packaging Applications

You can package your application using the Tizen IDE.

To package your application using the Tizen IDE:

  1. If you have made changes to the application after testing it, rebuild the application.

  2. Select the project in the Project Explorer view.

  3. Right-click the project and select Project > Build Package > TPK.

    You must sign your application package for its author to be identified and its integrity verified.

    If you have registered your certificate in the IDE, the signature is generated automatically during application packaging. Otherwise, the Signing pop-up is displayed. Register your certificate by clicking on the Preferences > Secure Profiles link displayed in the pop-up.

    If you do not have a certificate for signing and verifying your application, create it using the certificate generator.

  4. Click OK.

    After the packaging is complete, check the package in the project folder to ensure that the packaging was successful.

Packaging a Multi-project Application

Tizen supports the multi-project application that consists of the native service and UI application.

To package your application using Tizen IDE:

  1. In the IDE, make sure all the applications you want to include in the package are located in the Project Explorer view.

    To make the projects appear in the Project Explorer view, you can create new applications, or import existing projects or sample applications into the IDE.

  2. To establish a project reference between UI application and service application:

    • In the Project Explorer view, right-click the UI application.
    • Select Properties > Project References.
    • Select the check box for service application, and click OK.

      In the Project Explorer view, a message (with the UI application) appears next to the service application project name for all the applications you have added to the package.

After packaging the multi-project application, the packaging files contain the application binary, resource, and data files of the root and referenced applications. Their tizen-manifest.xml files are merged into one.
