Timeline Chart

The timeline chart shows the data values over time as a chart. You can create 12 different charts, which can be selected from the left side of the window.

Figure: Timeline chart

  1. CPU chart

    2. Shows CPU usage.

    • : System CPU usage (%)

    • : Application CPU Usage (%)

  2. CPU core chart

    Shows CPU core usage. Depending on the number of device cores, the number of lines on the chart varies.

    • : CPU core0 usage (%)

    • : CPU core1 usage (%)

    • : CPU core2 usage (%)

    • : CPU core3 usage (%)

    • : CPU core4 usage (%)

    • : CPU core5 usage (%)

    • : CPU core6 usage (%)

    • : CPU core7 usage (%)

  3. CPU frequency chart

    4. Shows CPU frequency. Depending on the number of device cores, the number of lines on the chart varies. In the Emulator, the frequency is always 0.

    • : CPU core0 frequency (KHz)

    • : CPU core1 frequency (KHz)

    • : CPU core2 frequency (KHz)

    • : CPU core3 frequency (KHz)

    • : CPU core4 frequency (KHz)

    • : CPU core5 frequency (KHz)

    • : CPU core6 frequency (KHz)

    • : CPU core7 frequency (KHz)

  4. Heap allocation chart

    5. Shows heap memory allocation. System total and app-allocated data is shown. Confirm additional binary allocation if you have set the binary in the setting dialog.

    If a process is added, a child chart is dynamically created for the added process.

    • : System total allocated (KB)

    • : App-allocated (KB)

    • Optionally, binary (user set) allocated (KB)
  5. Process Size chart

    Shows amount of memory usage. VSS is virtual memory, RSS is physical memory and PSS is process's private memory + shared memory / number of shared processes.

    If a process is added, a child chart is dynamically created for the added process.

    • : Total VSS (virtual set size) memory (KB) size of the process

    • : Physical RSS (resident set size) memory (KB) size used by the process

    • : PSS (proportional set size) memory (KB) in the following format: [(Unique memory + Shared memory)/number of processes sharing the memory].

  6. Memory chart

    Shows system memory usage.

    • : System maximum memory (KB)

    • : System used memory (KB)

    • : App used memory (KB)

  7. Screenshot chart

    Shows screenshots (they can be taken only when the Take snapshot button is enabled).

    8. Note
    The screenshot is captured about 2 seconds after the scene or form is transformed. To capture screenshots perfectly, use scene management or animation to transform a panel or form.

  8. Disk IO chart

    Shows the disk's read and write bytes and the number of sectors.

    • : Disk read operation counts

    • : Disk write operation counts

    • : Disk bytes read size (KB)

    • : Disk bytes written size (KB)

  9. Network IO

    Shows the network send and receive bytes.

    • : Receive bytes (KB)

    • : Send bytes (KB)

  10. UI Event chart

    Shows UI events that occur on the device.

    • : Touch event

    • : Gesture event

    • : Orientation event

  11. Device chart

    Shows the status of Wi-Fi, Bluetooth, GPS, DNet, and camera on the device.

    • : Wi-Fi

    • : Bluetooth

    • : GPS

    • : DNet

    • : Camera

  12. Energy chart

    Shows the power consumption of Flash memory, CPU, and LCD.

    • : Total energy usage (mA)

    • : Flash memory (mA)

    • : CPU (mA)

    • : LCD (mA)

Chart Buttons and Slider

You can use the chart buttons to ease the analysis.

Figure: Chart buttons

  1. Add button

    The button activates the Edit chart dialog. You can define which charts are visible in the Timeline page.

  2. Chart Minimize button

    Comparing different charts can be helpful. You can do it easily by minimizing all the charts.

  3. Chart Maximize button

    To see the chart in more detail, maximize it.

Managing Life-cycle and Scale

To manage the process life-cycle and modify the time scale:

  1. Confirm the status of the target process through the life-cycle bar.

    • : Initializing status

    • : Running status

    • : Pausing status

    • : Terminating status

  2. Adjust the chart's time scale using the scale slider at the bottom left.
