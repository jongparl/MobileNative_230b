<service-application> Element

The <service-application> element represents the manifest information of a Tizen native application without a graphical user interface (GUI).

<service-application> element

Manifest information of a Tizen native service application.

For more information on the relationship between the elements, see the element hierarchy.

Occurrences:

  • 1 or more (optional)

Expected children (in the following order):

Child element Occurrences
<label> 1 or more (optional)
<icon> 1 or more (optional)
<app-control> 1 or more (optional)
<metadata> 1 or more (optional)
<permission> 1 or more (optional)
<datacontrol> 1 or more (optional)

  • exec
  • multiple
  • nodisplay

appid attribute

Application unique ID. This can be used at launching or terminating the application explicitly.

Expected value:

  • String
exec attribute

Application executable file path.

Expected value:

  • String
multiple attribute

Indicates whether the application can be launched as multiple.

Expected value:

  • true
  • false
nodisplay attribute

Indicates whether the application is shown in the app-tray.

Expected value:

  • true
  • false
taskmanage attribute

Indicates whether the application is shown in the task manager.

Expected value:

  • true
  • false
enabled attribute

Indicates whether the application is enabled.

Expected value:

  • true
  • false
type attribute

Tizen application type.

Expected value:

  • capp
  • c++app
  • webapp
categories attribute

Categories the Tizen application belongs to.

Expected value:

  • String
extraid attribute

Extra ID for the Tizen application.

Expected value:

  • String
hw-acceleration attribute
Property based on which hardware rendering is used by the application.

Expected value:

  • use-GL
  • not-use-GL
  • use-system-setting
screen-reader attribute

Indicates whether the screen is enabled.

Expected value:

  • screen-reader-on
  • screen-reader-off
  • use-system-setting
recentimage attribute

Image for the Tizen application.

Expected value:

  • icon
  • capture
mainapp attribute

Indicates whether the executable file is the main application executable file for the Tizen package.

Expected value:

  • true
  • false
indicatordisplay attribute

Indicates whether the indicator display is enabled.

Expected value:

  • true
  • false
portrait-effectimage attribute

Portrait effect image or edj path.

Expected value:

  • String
landscape-effectimage attribute

Landscape effect image or edj path.

Expected value:

  • String
effectimage-type attribute

Effect image type before the application is launched.

Expected value:

  • String
guestmode-visibility attribute

Guest mode visibility for the Tizen application.

Expected value:

  • String
launchcondition attribute

Indicates whether the Tizen application has launch conditions.

Expected value:

  • true
  • false
permission-type attribute

Permission type for the Tizen application.

Expected value:

  • String
component-type attribute

Indicates whether the application is a UI or service application.

Expected value:

  • String
submode attribute

Indicates whether the sub mode is enabled for the Tizen application.

Expected value:

  • true
  • false
submode-mainid attribute

Sub mode ID for the Tizen application.

Expected value:

  • String
process-pool attribute

Indicates whether the application uses the process pool for the performance enhancement.

Expected value:

  • true
  • false
auto-restart attribute

Indicates whether the application is relaunched automatically if it is terminated abnormally.

Expected value:

  • true
  • false
on-boot attribute

Indicates whether the application is launched automatically on device booting time or application installation time.

Expected value:

  • true
  • false
multi-instance attribute

Indicates whether the application can have multi instance.

Expected value:

  • true
  • false
multi-instance-mainid attribute

ID for the multi instance.

Expected value:

  • String

For example:

<service-application appid="org.tizen.servicenew" auto-restart="false" exec="servicenew" multiple="false" 
                     nodisplay="true" on-boot="true" taskmanage="false" type="capp">
   <label>servicenew</label>
   <label xml:lang="eng-GB">testlabel</label>
   <icon>servicenew.png</icon>
   <app-control>
      <operation name="http://tizen.org/appcontrol/operation/compose"/>
      <mime name="application/pdf"/>
      <uri name="testuristring"/>
   </app-control>
   <metadata key="testkey1" value="testvalue2"/>
   <metadata key="testkey" value="testvalue"/>
</service-application>
